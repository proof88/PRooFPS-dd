 - Assuming Team Deathmatch manual test passes (_how-to-manual-test-01-team-deathmatch.txt) and that those
   functionalities are also working in Team Round Game mode, except 1 that is mentioned below separately.
   Therefore, not listing them here again.

 - Difference in Frag table compared to Team Deathmatch:
   - instead of Per-Team Total Frags, Per-Team Round Wins shall be shown next to number of players in team.
   - instead of Frag Limit, Round Win Limit shall be shown above the table.

 - Start a server instance with "Team Round Game" Game Mode, and 1 client instance (connect it to server), then test the following:
   - both players will stay in Spectator mode at the beginning. On the Team Select menu, select Spectator Mode on both instances.
   - check in both instances that Frag Table shows both players in the bottom table (Spectators), and both teams have 0 players.
   - check in both instances that neither the player objects nor their representation in the minimap are visible.
   - We assume that in the meantime Round game state has transitioned to Play state, so now restart the server from 'H' menu and
     quickly join the teams with both players:
     in Team Select menu, server joins team 1, client joins team 2.
   - Check that Round game states transition from Prepare -> Play -> WaitForReset during a round (kill players in different ways to see it),
   - Unless configured otherwise, Prepare state is 3 seconds, Play state is not limited in time, WaitForReset is 5 seconds.
   - Restarting the game zeros out not only player stats but also team round wins.
   - Reaching configured round win limit actually puts the game in won state and frag table is shown as being done also in other game modes.
   - Disconnect and then reconnect the client when the server is in WaitForReset state, check if client immediately replicates
     this state and allows all movements and actions in such state.
   - Disconnect and then reconnect the client when the server is in Prepare state, check if client immediately replicates
     this state and does not allow any movement in such state (except crouch, inventory item use).
   - Put the client player into Spectator Mode. The current round shall end without round win.
   - Wait for the next round. With the server player being the only player, kill the player: the round state shall transition to WaitForReset
     but no round win shall be registered due to the other team is empty.
   - Disconnect the client.
   - With the server player being the only player, kill the player: the round state shall transition to WaitForReset but no round win
     shall be registered due to the other team is empty.

   - TBD: Reconnect the client when the round is in Play state and join a team. The client shall stay in force-spectating state (dead) until
     the next round begins.
   - TBD: Reconnect the client when the round is in WaitForReset state and join a team. The client shall stay in force-spectating state (dead) until
     the next round begins.
